## 1. 目的・成功条件

**目的**  
自分とこゆたん向けに、東京23区のデートプラン提案・お店／場所選びを支援する対話Bot。

**成功条件**
- Colab上で LoRA（スロット抽出）＋ Gemini（案3つ＋理由）が動く
- 学習済み LoRA を Drive に保存し、再学習なしで再開できる
- 公開GitHubにコード・docs・サンプルデータを載せる（秘密情報なし）

---

## 2. 利用者・使い方

| 項目 | 決定内容 |
|------|----------|
| 利用者 | 自分とこゆたん（呼称: リョウスケ君 / こゆたん） |
| 入力 | 複数ターン対話 |
| 出力 | **案を3つ**、それぞれに**理由つき**（Gemini） |
| 口調 | フランク（Gemini側） |
| 失敗時 | 条件不足は定型で聞き返し／Gemini失敗は簡易フォールバック |

---

## 3. 対象範囲

| やる | やらない |
|------|----------|
| デートプラン提案 | 予約代行 |
| エリア＋好みに基づく店・スポット提案 | 空席／在庫のリアルタイム確認 |
| 東京23区（優先: 渋谷・上野・新宿／お台場〜品川） | 恋愛相談の深掘り |
| 安価（学生向け）を優先 | 個人情報の取り扱い |
| 店舗マスタCSVは使わない | |

**可変条件（対話で蓄積）**: budget / time_slot / area / mood / avoid_areas  
**デフォルト予算**: 未指定時 一人 3000円

---

## 4. データ・モデル（現行）

| 項目 | 決定内容 |
|------|----------|
| ベースモデル | `Qwen/Qwen2.5-1.5B-Instruct`（OOM時は 0.5B） |
| LoRA | r=8, target=`q_proj,k_proj,v_proj,o_proj` |
| LoRAの役割 | **スロット抽出のみ**（方針文は出さない） |
| 学習データ | `date_bot_train_slots.csv`（input/output、約80件想定） |
| Gemini | `gemini-3.1-flash-lite`、**REST直叩き**（SDK不使用） |
| Geminiへの入力 | **slots + 好み要約のみ**（方針文なし） |
| 店舗リスト | なし |
| 個人情報 | 一切扱わない。LINEは要約のみ。生ログ非公開 |

### LoRA出力フォーマット

```text
budget:3000
time_slot:昼/午後
area:上野
mood:コスパ
avoid_areas:新宿
```

---

## 5. 処理パイプライン

```text
発話
  → LoRA でスロット文字列生成
  → parse（リテラル\n対応）
  → postprocess（幻覚抑制・予算ルール補完）
  → slots にマージ（累積。resetでクリア）
  → 不足（time_slot / area）なら定型聞き返し
  → Gemini REST で案3つ＋理由
  → 失敗時はエリア別ルールベース案
```

## 8. 受け入れ基準（現行）

1. 複数ターンで slots が蓄積される  
2. 条件が揃うと案3つ＋理由が出る  
3. 安価・23区が提案に反映される  
4. 予約代行・在庫・恋愛深掘りに乗らない  
5. LoRAを Drive 保存し、次回 `PeftModel.from_pretrained` で再開できる  
6. 公開リポジトリに秘密情報・生ログがない  

---

## 9. 既知の注意点

- 変数名 `model` を Gemini 診断ループで上書きしない（`cand_name` を使う）
- `generate` は `use_cache=False`（DynamicCache エラー回避）
- 短い地名（渋谷等）は短文スキップの対象外にする（ノート改訂時に反映）
- LoRA単体の exact_match は低め → **後処理必須**。評価は `date_bot_eval_gold.csv` で実施

以上を合意内容とする。
